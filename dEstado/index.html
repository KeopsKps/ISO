Nomenclatura

Columna Significado
Estado Nombre del estado
Transición de entrada Transición que se llevo acabo antes que se llegara al estado
Descripción estado Breve descripción del estado
Transición de salida Descripción de que ocurre para que finalize el estado y pase al siguiente

Descripción de los estados del Diagrama General de Sistema Web

Estado Transición de entrada Descripción estado Transición de salida
Ingreso al Sistema Ingreso pagina web. Estado de acceso al sistema web, para consultar tramite o solicitar uno nuevo. Se acepta el ingreso al sistema.
Verificación Ingreso al sistema. Verificación de los datos enviados como ingreso al sistema. Se verifican los datos de entrada al sistema.
Solicitud Verificación de los datos de entrada al sistema. En el caso ideal después que se realiza la verificación el usuario procede a realizar la solicitud de tramite, ingresando todos los datos necesarios. Se terminan de llenar los datos de la solicitud y se genera la misma.
PreSolicitud Ingreso pagina web. Estado alternativo al sistema en el que primero se realiza la PreSolicitud del tramite. Se terminan de llenar los datos necesarios.
Registro PreSolicitud Terminación de datos de entrada de PreSolicitud. Registro de datos de PreSolicitud, estos datos deben ser llenados sin necesidad de tener una cuenta. Finalización del registro de la PreSolicitud.

Descripción de los estados del Diagrama General de Sistema de Escritorio

Estado Transición de entrada Descripción estado Transición de salida
Informe Ingreso al sistema de entrada. Estado que solo sirve para tratar con los informes. Se genera los informes disponibles.
Tramite Ingreso al sistema de entrada. Ruta alternativa a los informes y se refiere a ver las solicitudes de tramites nuevas. Selecciona un tramite para verificación.
Requisitos Selección de tramite. Seleccionado un requisito se verifica que cumpla con los requisitos establecidos. Si se cumplen los datos se piden los datos que faltan sino se vuelve al estado Tramite.
Datos Correcta verificación de los datos. Solicitud de los datos que hacen falta para completar el tramite. Se guardan los datos del tramite.
Registro Datos completos del tramite. Se guardan los datos completos del tramite. Finalización de guardado de datos del tramite.
Proyecto Si el tramite registrado es un proyecto Se solicitan los datos necesarios de un proyecto. Ingreso de datos necesarios del proyecto.
RegistroProyecto Datos necesarios especificos del proyecto. Se guardan los datos del proyecto, asi como la creación del mismo Creación de un proyecto.
